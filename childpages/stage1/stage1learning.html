Style

We can style our elements, not one by one, but by combining them in different ways. Either through their parent group, or by different tags we assign them (for example: class or id).          CSS helps us with that.
We can make one CSS file that we use across all the different html pages, (but it sure needs discipline).

Semantics

Here I am back to write a bit more about semantics.
Certain HTML elements have been included into html5 to increase its semanticisity. This means that they try to express what is inside them, instead of just saying how they should look. (example <b> exchanged for <strong>, because the second one explains more). One should take care to name ones classes also in a semantic manner, thus what is its role, its meaning, and not its looks. Some semantic elements now are <section>, <footer>, <article>, and they've been chosen by looking into which classes people used the most.

Something about trees

Websites have a tree like structure
which means that there are parent elements that contain other elements and can themselves be contained by even "bigger" ones.

On wikipedia they have really many visual representations of how tree structures interact.


It is interesting how this nested elements interact or what does it actually mean.
Some elements inherit characterisitcs, some elements divide the space.
